## Что чиним

### 1. Дёргающийся «код» на главной (HowItWorks)

На скриншоте видно, что строка ломается и показывает мусор вида `https:var(--accent))">"color:hsl(var(--muted-foreground))">//yoursit`. Это баг подсветки в `src/components/ui/typing-code-block.tsx`:

- В `highlight()` сначала запускается регекс комментариев `\/\/[^\n]*`, который оборачивает `//yoursite.ru` в `<span style="color:hsl(var(--muted-foreground))">…</span>`.
- Затем по результату прогоняется регекс строк `(["'`])…\1` — он находит кавычки **внутри атрибута `style="…"`** уже вставленного span и портит разметку.
- На мобиле строка длиннее ширины экрана → перенос → визуальный «прыжок» layout при каждой новой букве.

Что делаем:

- Перепишем `highlight()` на токенизацию: один проход по строке с приоритетом comment > string > keyword > number > mark, без повторных regex-проходов по уже вставленному HTML. Это полностью убирает мусор.
- Для `variant="inline"` зарезервируем фиксированную высоту (одна строка моноширинного шрифта) и добавим `overflow-hidden text-ellipsis whitespace-nowrap` в обёртку — чтобы при печати символов layout не дёргался.
- В `HowItWorks.tsx` обернём `<TypingCodeBlock>` в контейнер `min-h-[28px] max-w-full overflow-hidden` с `mt-10 flex justify-center`.

Что меняем в `src/components/landing/ComparisonSection.tsx`:

**A. Layout — единая таблица.**

Удаляем мобильный блочный рендер (`md:hidden space-y-2`). Оставляем одну таблицу для всех экранов, но компактную:

- Сужаем колонку «Функция» (короткие названия, hint скрываем на мобиле через `hidden sm:block`).
- Имена конкурентов на мобиле — короткие алиасы (`OWN / SR / SF / АРС`) с полным названием через `<abbr title>`; на ≥ sm — полные.
- Высота строк уменьшается, иконки `Cell` становятся `w-6 h-6` (на мобиле `w-5 h-5`).
- Горизонтальный скролл таблицы только если контент не помещается: `overflow-x-auto` на обёртке.
