## Что происходит на самом деле

Страницы `/geo-rating`, `/geo-audit`, `/site-formula`, `/tools` **открываются и работают** — я только что проверил это в браузере. Скриншот десктопа показывает полностью отрендеренный React-интерфейс GEO Рейтинга.

Проблема не в роутинге и не в моих правках. Проблема — это **вспышка статического SEO-фолбэка** из `index.html` (тот самый "голый HTML" без стилей: `OWNDEV / Инструменты · GEO-аудит / GEO и AI-ready аудит сайта для ChatGPT…`), которая видна 1–5 секунд, пока React не смонтировался. На медленных страницах (`/geo-rating` тянет 92 сайта, `/tools` рендерит большой каталог, `/site-formula/v3` — тяжёлый wizard) этот фолбэк остаётся на экране дольше, и выглядит как "страница сломалась".

Я воспроизвёл это на мобильном вьюпорте: первые секунды виден чистый HTML без CSS, через ~5 сек появляется нормальный React-интерфейс (тёмная тема, таблица рейтинга, фильтры).

Это в точности тот фикс, который мы уже обсуждали в прошлом сообщении ("flash при старте"), но патч не успел применился — в `index.html` нет блока `ssr-fallback-hide`. Делаю его сейчас.

## План

### 1. `index.html` — спрятать SEO-фолбэк от живых пользователей
В `<head>` добавить inline `<style id="ssr-fallback-hide">`, который скрывает статический `header/main/footer` внутри `#root` для браузеров:

```html
<style id="ssr-fallback-hide">
  #root > header,
  #root > main,
  #root > footer { display: none !important; }
  html, body { background: #0f172a; color: #e5e7eb; margin: 0; }
</style>
```

- Поисковые/AI-боты (GPTBot, PerplexityBot, ClaudeBot, ЯндексBot) **продолжат видеть фолбэк** — они не рендерят CSS, а парсят HTML напрямую → SEO/GEO не пострадает.
- Живые пользователи увидят тёмный фон вместо белой "голой" вёрстки, пока React монтируется.
- Когда React делает `createRoot(...).render(<App/>)`, `#root > header/main/footer` исчезают полностью, селектор больше ни на что не действует.

### 2. Ничего больше не трогаю
- Роуты в `App.tsx` (`/geo-rating`, `/geo-audit`, `/site-formula/*`, `/tools/*`) — на месте и работают.
- `Header.tsx`, навигацию, мобильное меню — не трогаю (защищено memory-правилом).
- Бэкенд, темизацию Site Formula PRO (предыдущие правки в `ProReportPanel.tsx` и `SiteFormulaV3.tsx`) — оставляю как есть.
