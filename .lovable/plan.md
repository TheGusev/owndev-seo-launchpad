

## Усиление Direct-модуля до платного продукта

### Фаза 1 — Быстрые улучшения (без AI-вызовов)

**Файл: `supabase/functions/site-check-scan/index.ts`**, функция `directAudit`:

#### 1. Проверка Яндекс.Метрики (новая проверка #7)
Поиск в HTML счётчика Метрики (`mc.yandex.ru/metrika/tag.js` или `ym(NNNN`). Если нет — critical issue: «Нет Яндекс.Метрики — невозможно отследить конверсии из Директа».

#### 2. Проверка мобильного viewport (новая проверка #8)
Проверка `<meta name="viewport">`. Если нет или неправильный — high issue: «Страница не адаптирована для мобильного трафика Директа».

#### 3. UTM-совместимость (новая проверка #9)
Проверка наличия canonical с query-параметрами (ломает UTM). Если canonical жёстко фиксирован без учёта UTM — medium issue.

#### 4. Оценка «Direct Readiness Score» 
Добавить в `DirectAuditResult` поле `readiness_score: number` (0-10). Рассчитывается по количеству пройденных/проваленных проверок с весами. Выводится как визуальная шкала в UI.

### Фаза 2 — AI-генерация объявления (новый AI-вызов)

**Файл: `supabase/functions/site-check-scan/index.ts`**, в основном pipeline после `directAudit`:

#### 5. Генерация полного объявления через AI
Вызов Lovable AI Gateway с промптом на основе H1, Title, текста страницы и темы. Результат:

```typescript
interface DirectAdSuggestion {
  headline1: string;      // до 35 символов
  headline2: string;      // до 30 символов
  ad_text: string;        // до 81 символа
  sitelinks: { title: string; description: string }[];  // 4 быстрые ссылки
  callouts: string[];     // 4 уточнения
}
```

Это добавляется как отдельный шаг между Step 3 и Step 4 (или в конец Step 3).

### Фаза 3 — UI для отображения

**Файл: новый `src/components/site-check/DirectAdPreview.tsx`**

Компонент-превью сгенерированного объявления в стиле Яндекс.Директа:
- Визуально выглядит как реальное объявление в поиске
- Кнопки «Скопировать заголовок», «Скопировать всё»
- Блок с быстрыми ссылками и уточнениями
- Шкала «Direct Readiness: 8/10»

**Файл: `src/components/site-check/ScoreCards.tsx`** (или где отображаются скоры)

Добавить карточку readiness_score рядом с direct_score.

### Что НЕ меняется
- Существующие 6 проверок остаются
- Все текущие скоры и логика нетронуты
- Keywords и minus words остаются в своих шагах

### Порядок реализации
1. Фаза 1 (3 новые проверки + readiness_score) — без дополнительных AI-вызовов, быстро
2. Фаза 3 (UI превью) — отображение нового формата
3. Фаза 2 (AI-генерация объявления) — самая ценная часть, добавляет задержку ~3-5 сек к скану
