

## Цель

Перебалансировать финальную формулу итогового скора в `calcScoresWeighted`, снизив вес `schema` и подняв вес `seo` и `ai`.

## Файл

`owndev-backend/src/services/SiteCheckPipeline.ts` — единственная правка.

## Правка

Строка 439:

**Было:**
```ts
const total = Math.round(seo * 0.3 + direct * 0.2 + schema * 0.25 + ai * 0.25);
```

**Станет:**
```ts
const total = Math.round(seo * 0.35 + direct * 0.20 + schema * 0.15 + ai * 0.30);
```

Сумма коэффициентов = 1.0, итог остаётся в диапазоне 0–100. Округление через `Math.round` сохраняется.

## Что НЕ трогаем

- Сами функции подсчёта `seo`, `direct`, `schema`, `ai` блоков (строки 384–438) — без изменений.
- `DIRECT_WEIGHTS`, `DIRECT_CHECKS_META`, `directAudit`, фронтовая визуализация скора, БД-схема `geo_rating` (поля те же), правила памяти, маршруты.

## Побочный эффект

- У сайтов с богатой schema-разметкой итоговый скор слегка просядет (было 25% → стало 15%).
- У сайтов с хорошим SEO и AI-готовностью (FAQ, llms.txt, структурированный контент) — наоборот вырастет.
- При следующем массовом ресканировании числа в `geo_rating.total_score` пересчитаются автоматически. Старые записи (без рескана) останутся с прежним балансом до очередного аудита.

## Проверка

1. `cd owndev-backend && npx tsc --noEmit` — без ошибок.
2. Тестовый запуск site-check на сайте с `seo=80, direct=50, schema=20, ai=70` → старый total = 24+10+5+17.5 = 56.5 → 57; новый total = 28+10+3+21 = 62. Числа сходятся.
3. На фронте `/site-check` итоговый скор отрисовывается из `scores.total` — никаких правок в UI не требуется.
