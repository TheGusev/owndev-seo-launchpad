

## Страница методологии GEO рейтинга `/geo-rating/methodology`

### Что создаём

Новая страница с детальным описанием всех сигналов, по которым OWNDEV оценивает сайты. Данные берём из реального `AuditService` — он проверяет 6 блоков с конкретными проверками.

### Реальные сигналы из AuditService (confirmed by code)

| Блок | Вес | Проверки |
|------|-----|----------|
| Indexability | 15% | noindex meta, X-Robots-Tag, robots.txt block, canonical mismatch |
| Content Structure | 20% | H1 (отсутствие / дубли), H2, объём текста (<300 слов) |
| AI Readiness | 20% | llms.txt, FAQ-секции, списки/таблицы |
| Schema.org | 15% | JSON-LD наличие, рекомендуемые типы, @type, name/headline |
| E-E-A-T | 15% | Контакты, соцсети, авторство |
| Technical | 15% | HTTP-статус, скорость (>3s), размер HTML (>200KB) |

Итого ~20 конкретных проверок. Плюс CrawlerService собирает мета-данные (robots, canonical, links, schemas, headers) — это ещё ~10 сигналов на входе. Плюс GEO-рейтинг добавляет агрегатные: llms.txt наличие, schema наличие, FAQ наличие, LLM Score, SEO Score, errors count — ещё ~6. Итого 35+ конкретных + методология подсчёта = "50+ сигналов" обосновано.

### Структура страницы

1. **Hero**: заголовок "Методология GEO Рейтинга", подзаголовок, breadcrumb назад
2. **Общий принцип**: формула score = средневзвешенное по 6 блокам, таблица весов
3. **6 секций-аккордеонов** — по одной на каждый блок:
   - Название блока + вес + иконка
   - Таблица проверок: что проверяется, severity, приоритет, рекомендация
4. **Scoring**: как считается blockScore (penalty-система: P1=-30, P2=-15, P3=-5)
5. **Агрегация для рейтинга**: как LLM Score и SEO Score маппятся на итоговый ранг
6. **CTA**: "Проверить свой сайт"
